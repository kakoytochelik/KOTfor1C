# Блок: Infobase Manager

## Что это

`KOT Infobase Manager` — отдельная webview-панель для управления базами 1С, которые расширение использует в разных сценариях:

- built-in запуск Vanessa;
- запуск и установка `KOT Form Explorer`;
- служебные runtime-ИБ расширения;
- ручные пользовательские базы;
- launcher/server/web записи, которые KOT умеет переиспользовать как цели запуска.

Менеджер нужен как единая точка работы с базами, чтобы один и тот же список ИБ и их метаданные переиспользовались в picker-ах `Vanessa` и `Form Explorer`.

Важно: destructive-операции (`create`, `copy`, `recreate`, `restore DT`, `delete`, перенос каталога) доступны только для файловых ИБ. Серверные и web-записи тоже попадают в список, но используются в первую очередь как цели запуска и как общий справочник подключений.

## Откуда берется список баз

В панели объединяются несколько источников:

- `launcher` — базы, найденные в `ibases.v8i` лаунчера 1С;
- `runtime` — служебные ИБ и runtime-каталоги, которые создает само расширение;
- `manual` — пути, которые пользователь добавил вручную;
- `snapshot` — базы, восстановленные из source-информации в snapshot-ах `Form Explorer`;
- `workspaceState` — базы и метаданные, которые расширение помнит в состоянии текущего workspace.

Одна и та же база может одновременно иметь несколько источников. Например, база может быть и в `launcher`, и в `workspaceState`, и при этом участвовать в `snapshot`.

## Какие бывают roles

`Roles` показывают не то, где база лежит, а в каком сценарии KOT ее использует или уже использовал.

- `startup` — общая легковесная startup-ИБ расширения для служебных запусков;
- `vanessa` — база использовалась или была подготовлена для built-in запуска Vanessa;
- `formExplorer` — база участвует в сценариях `KOT Form Explorer` (builder-ИБ или целевая база установки/запуска);
- `snapshot` — база была замечена как источник уже существующего snapshot-а `Form Explorer`.

Роли назначаются автоматически:

- по факту обнаружения известных runtime-путей расширения;
- по данным snapshot-а `Form Explorer`;
- по сохраненным метаданным последнего запуска/подготовки;
- по сохраненным пользовательским выборам базы для Vanessa/Form Explorer.

`Roles` не являются ручной настройкой. Это вычисляемые признаки, которые помогают понять, почему база попала в список и где она уже использовалась.

## Какие бывают sources

`Sources` показывают, из какого канала расширение вообще узнало о базе.

- `launcher` — база пришла из `ibases.v8i`;
- `runtime` — база лежит в известном runtime-каталоге KOT;
- `manual` — путь был добавлен пользователем вручную;
- `snapshot` — база восстановлена из snapshot JSON;
- `workspaceState` — база известна только по сохраненному состоянию workspace, даже если сейчас не найдена в launcher.

Практический смысл:

- `sources` отвечают на вопрос: «почему эта база видна в менеджере»;
- `roles` отвечают на вопрос: «для чего KOT ее использует».

## Что умеет панель

Из одной панели доступны:

- запуск базы в `1C:Enterprise`;
- запуск базы в `Designer`;
- запуск выбранной базы через `Start Form Explorer` с последующим открытием панели `KOT Form Explorer` (кроме web ИБ);
- настройка per-base ключей запуска 1С;
- редактирование отображаемого имени базы;
- добавление/удаление записи в launcher 1С;
- открытие папки базы и связанных логов;
- создание новой файловой ИБ с регистрацией в launcher 1С;
- копирование файловой ИБ в другой каталог с регистрацией копии в launcher 1С;
- пересоздание существующей файловой ИБ с optional backup в `DT`;
- восстановление файловой ИБ из `DT`;
- выгрузка файловой ИБ в `DT`;
- сохранение текущей конфигурации файловой ИБ в `.cf`;
- обновление конфигурации файловой ИБ из `kotTestToolkit.formExplorer.configurationSourceDirectory` или из выбранного `.cf`;
- перенос или переименование каталога файловой ИБ;
- переназначение item-а на другой уже существующий каталог файловой ИБ без физического переноса файлов;
- полное удаление файловой ИБ: файлы + launcher + сохраненные ссылки менеджера.

## Сортировка списка баз

В боковом списке доступны три режима сортировки:

- `Last opened` — по дате последнего запуска базы из KOT;
- `Alphabetical` — по имени базы;
- `Manual order` — пользовательский порядок.

В `Manual order` можно перетаскивать базы мышью. Порядок сохраняется в `workspaceState`, поэтому переживает переключение режима сортировки, повторное открытие панели и перезапуск VS Code.

## Ключи запуска

Для каждой базы можно задать свои launch keys.

Логика такая:

- по умолчанию база запускается без дополнительных ключей;
- если для базы указаны свои ключи, при запуске из менеджера и при `Start Form Explorer` используются они;
- отдельно можно выбрать режим `Use workspace defaults`, тогда применяется общий `kotTestToolkit.startupParams.parameters`;
- если запуск идет без заранее известных credentials, менеджер убирает подавление startup dialogs, чтобы 1С могла показать логин-пароль пользователю.

## Безопасность destructive-операций

Перед опасными действиями менеджер:

- просит отдельное подтверждение;
- проверяет, не занята ли файловая база;
- может сделать backup в `DT` перед `recreate` или полным `delete`.

Это не заменяет полноценные проектные backup-практики, но заметно снижает риск случайно уничтожить рабочую ИБ.

## Как менеджер связан с Vanessa и Form Explorer

- built-in запуск Vanessa использует тот же слой выбора и хранения ИБ;
- `KOT Form Explorer` использует тот же список и те же метаданные;
- при запуске из менеджера через `Start Form Explorer` выбранная база сразу передается в сценарий установки/старта адаптера;
- после операций менеджера обновленные имя, путь, launcher-registration и launch keys становятся доступны и в остальных picker-ах расширения.
