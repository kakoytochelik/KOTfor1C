# Блок: Navigation and File Operations

## Задача блока

Ускорить переходы между сценариями и упростить работу с связанными файлами (MXL и пути в проекте).

## Команды навигации

### `KOT - Open scenario` (`KOT - Открыть сценарий`)

- Открывает вызываемый вложенный сценарий по текущей строке.
- Удобно при разборе длинных цепочек вызовов.

### `KOT - Find references to current scenario` (`KOT - Найти вызовы текущего сценария`)

- Показывает все места, где вызывается текущий сценарий.
- Удобно для анализа влияния перед правками.

## Подсветка связанных тестов

- Когда открыт сценарий (главный или вложенный), в Test Manager (`Менеджер тестов`) подсвечиваются связанные группы и главные сценарии по цепочке вызовов.
- Это помогает быстро понять, какие тесты затронуты текущими изменениями.

## Работа с файлами по выделенному тексту

### `KOT - Open MXL file in editor` (`KOT - Открыть MXL файл в редакторе`)

- Ищет MXL-файл по выделенному имени.
- Открывает через [1С:Предприятие — работа с файлами](https://v8.1c.ru/static/1s-predpriyatie-rabota-s-faylami/).

### `KOT - Reveal file in VS Code Explorer` (`KOT - Показать файл в проводнике VS Code`)

- Находит файл в проекте и фокусирует его в панели Explorer.

### `KOT - Reveal file in OS file manager` (`KOT - Показать файл в системном проводнике`)

- Открывает системный проводник с выделением/переходом к файлу.

## Настройки блока

| Настройка | Назначение |
|---|---|
| `kotTestToolkit.paths.fileWorkshopExe` | Путь к `1cv8fv.exe` для MXL-команд |
| `kotTestToolkit.paths.yamlSourceDirectory` | Область поиска сценариев |
| `kotTestToolkit.phaseSwitcher.highlightAffectedMainScenarios` | Подсветка связанных групп и главных сценариев для открытого файла |

## Ограничения

- Без корректного `fileWorkshopExe` MXL-команды не работают.
- На macOS MXL-функция недоступна из-за отсутствия клиента [1С:Предприятие — работа с файлами](https://v8.1c.ru/static/1s-predpriyatie-rabota-s-faylami/).
