# Блок: Test Manager

## Задача блока

Test Manager (`Менеджер тестов`, бывш. Phase Switcher) управляет тем, какие главные сценарии участвуют в сборке и запуске, и показывает текущее состояние по каждому тесту.

## Что пользователь видит
- Верхнюю панель быстрых действий:
  - `Open Vanessa`;
  - `Open KOT Form Explorer`;
  - `Open Infobase Manager`;
  - `More actions`.
- Меню `More actions` (кнопка `...`) с быстрыми действиями:
  - `Reload tests from disk`;
  - `Refresh Vanessa steps library`;
  - `Repair scenario files` с подкомандами `Repair changed scenarios` и `Repair all scenarios`;
  - `Cancel scenario repair`;
  - `Open Build Scenario Parameters Manager`;
  - `Manage platforms`;
  - `Open extension settings`.
- Вкладки `Main` (`Главные`) и `Favorites` (`Избранное`).
- Панель управления тестами:
  - создание главного или вложенного сценария;
  - поиск по главным сценариям;
  - `Collapse/Expand all`;
  - `Toggle all checkboxes`;
  - `Defaults`.
- Дерево групп и тестов:
  - чекбоксы выбора;
  - кнопка запуска;
  - кнопка открытия файла сценария;
  - для главных сценариев: открытие `test.yaml`, built/combined JSON, переименование и удаление;
  - для `failed` сценариев — кнопка быстрого перехода к упавшему шагу в `feature`.
- Чекбоксы включения/выключения тестов.
- Индикатор прогресса прогона (проценты, при наведении — счетчик `x/y`).
- Строку статуса `Will be built: X/Y`.
- Split-button `Build tests` (`Собрать тесты`) с выпадающим меню опций.
- Кнопку `Cancel build` (`Отменить сборку`).

## Основные возможности

### Управление группами и тестами

- Включение/выключение тестов поштучно.
- Массовое включение по группе.
- Выбор чекбоксов сразу сохраняется и используется при сборке.
- Переименование группы через контекстное меню.
- Для главных сценариев: открытие `test.yaml`, `Open JSON -> Open built JSON / Open combined JSON`, переименование и удаление через контекстное меню.

### Сборка из Test Manager

- Основное нажатие `Build tests` сразу запускает сборку.
- В выпадающем меню `Build tests` можно быстро переключать активный профиль Менеджера параметров.
- `Accounting mode` показывается в этом меню только при включенных 1C:Drive-специфичных элементах UI.

### Поиск по сценариям

- Включается кнопкой с лупой в шапке Test Manager (`Менеджер тестов`).
- Подсветка совпадений активна, пока фокус находится в поле поиска.
- Работает только по списку `Main`; на вкладке `Favorites` вместо поиска доступна сортировка.
- Во время поиска подсветка связанных сценариев временно отключается, чтобы не мешать навигации.

### Статусы запуска

Для каждого сценария отображается актуальный status:

- `running` — тест в процессе;
- `passed` — успешное завершение. Дополнительно имеет stale состояние, если запускался устаревший тест;
- `failed` — ошибка выполнения;
- `stale` — сборка устарела (после изменений в связанных сценариях).

Поведение строки сценария в зависимости от статуса:

- `running`:
  - вместо чекбокса отображается спиннер;
  - кнопка запуска скрыта;
  - клик по строке открывает `feature` на текущем выполняемом шаге.
- `failed`:
  - доступна кнопка перехода к упавшему шагу в `feature`;
  - клик по строке открывает исходный `yaml` сценария.

### Избранное

- Добавление/удаление сценария в избранное.
- Сортировка списка по коду или имени.
- Открытие сценария по клику.
- Drag-and-drop в редактор для вставки вызова вложенного сценария с параметрами.

### Подсветка влияния

Если открыт сценарий в редакторе, Test Manager (`Менеджер тестов`) может подсвечивать главные сценарии, на которые он влияет по цепочке вызовов.

## Настройки блока

| Настройка | Назначение |
|---|---|
| `kotTestToolkit.phaseSwitcher.highlightAffectedMainScenarios` | Включает подсветку затронутых главных сценариев |
| `kotTestToolkit.phaseSwitcher.autoAddNewScenariosToFavorites` | Автодобавление новых сценариев в избранное |
| `kotTestToolkit.assembleScript.showDriveFeatures` | Показать/скрыть 1C:Drive-специфичные элементы (`Build FL` и режим учета) |

## Типовые операции

1. Выбрать набор тестов по чекбоксам.
2. При необходимости открыть выпадающее меню `Build tests` и переключить профиль параметров.
3. Нажать `Build tests` (`Собрать тесты`).
4. Запустить конкретный сценарий и смотреть статус/лог.

## Что важно учитывать

- Кнопки запуска зависят от артефактов последней сборки.
- Built-in запуск Vanessa использует `combined JSON`, а исходный built JSON остается доступен для просмотра и сравнения.
- После изменений в сценариях успешный/ошибочный статус может перейти в `stale`.
- Во время `running` повторный запуск этого же сценария блокируется.
- Во время выполнения хотя бы одного теста блокируются `Build tests` и чекбоксы выбора сценариев.
- `Expand/Collapse all` и разворачивание/сворачивание групп остаются доступными во время прогона.
- Для фильтрации сборки расширение автоматически подставляет `Exceptscenario` или `Scenariofilter` по выбранным чекбоксам.
