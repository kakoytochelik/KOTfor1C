# Блок: Etalon bases

## Что это

`Etalon bases` — это отдельный каталог эталонных баз в `bases.yaml`, который KOT использует для работы с `test.yaml`, подготовкой тестовых баз и автоматическим разрешением профилей пользователей.

Это не список рабочих ИБ из `Infobase Manager`, а справочник эталонных баз проекта.

## Где хранится каталог

- Путь к каталогу задается параметром СППР `ModelDBSettings` в Менеджере параметров.
- Если путь не задан явно, KOT использует workspace-local default `.vscode/kot-runtime/bases.yaml`.
- Относительные пути резолвятся относительно корня workspace.

Подробнее о связи с параметрами СППР: [parameters-manager.md](./parameters-manager.md)

## Что хранится в одной записи

Запись эталонной базы содержит:

- отображаемое имя базы;
- `ИдентификаторБазы`;
- путь к `DT`;
- список профилей пользователей;
- для каждого профиля: `ПрофильПользователя`, логин и пароль.

Если путь к `DT` относительный, он резолвится относительно папки самого `bases.yaml`.

## Как открыть менеджер эталонных баз

- Команда `KOT - Manage etalon bases` (`KOT - Управление эталонными базами`).
- Кнопка `Etalon bases` в `Infobase Manager`.
- Из некоторых сценариев создания и редактирования тестов KOT может предлагать открыть этот менеджер, если каталог еще пуст.

## Что умеет менеджер

- добавлять новую эталонную базу;
- редактировать имя, идентификатор и путь к `DT`;
- удалять запись;
- управлять профилями пользователей для конкретной базы;
- импортировать данные из внешнего `bases.yaml`;
- экспортировать текущий каталог во внешний `bases.yaml`;
- открывать текущий `bases.yaml`.

## Где `Etalon bases` используются в KOT

### Создание главного сценария

При создании главного сценария KOT может:

- предложить выбрать эталонную базу;
- затем предложить профиль пользователя из выбранной базы;
- записать эти значения в `test.yaml`.

Подробнее: [scenario-creation.md](./scenario-creation.md)

### Редактирование `test.yaml`

В `test.yaml` эталонные базы используются для быстрых действий по карандашу:

- смена `ЭталоннаяБазаИмя` / `ИдентификаторБазы`;
- выбор `ПрофильПользователя` из профилей выбранной эталонной базы.

### Built-in запуск Vanessa

При подготовке базы перед built-in запуском Vanessa KOT может предложить вариант:

- `Restore from etalon DT`

В этом случае путь к `DT` берется из `bases.yaml` по данным `test.yaml` и `ModelDBSettings`.

### Совместимость со СППР

Каталог `Etalon bases` помогает СППР:

- разрешать путь к `DT` эталонной базы;
- разрешать логин и пароль `TestClient` по `ПрофильПользователя`;
- работать ближе к привычной модели `ModelDBSettings`.

## Что важно учитывать

- Если `bases.yaml` пустой, KOT не блокирует создание главного сценария: поля `test.yaml` можно заполнить позже вручную.
- Если `ModelDBSettings` пустой, KOT не сможет использовать каталог эталонных баз автоматически.
- При `AuthCompile=true` корректный `ModelDBSettings` становится обязательным для строгой валидации СППР.
- `Etalon bases` и рабочие ИБ из `Infobase Manager` — это разные сущности, даже если они относятся к одной и той же реальной базе.
