# Блок: Platform Manager

## Что это

`Platform Manager` — это каталог доступных платформ 1С, которые KOT использует при запуске баз, `KOT Form Explorer` и built-in запуска Vanessa.

KOT хранит не просто версию платформы, а конкретный путь к клиенту `1cv8c.exe` и отображаемое имя записи.

## Как открыть

- Команда `KOT - Manage 1C platforms` (`KOT - Управление платформами 1С`).
- Меню `...` в `Test Manager` -> `Manage platforms`.
- Ссылка из настройки `kotTestToolkit.platforms.catalog`.

## Что хранится в каталоге

Каждая запись содержит:

- `name` — отображаемое имя платформы;
- `clientExePath` — путь к `1cv8c.exe`.

Важно:

- первая запись в списке считается платформой по умолчанию;
- именно эта платформа используется там, где KOT не получил более конкретный выбор;
- если путь указывает на `1cv8.exe`, KOT пытается нормализовать его до sibling `1cv8c.exe`.

## Автообнаружение платформ

- Если каталог платформ пуст, KOT пытается автоматически найти установленные платформы в стандартных Windows-путях.
- Найденные версии добавляются в каталог автоматически.
- Самая свежая обнаруженная версия ставится первой и становится платформой по умолчанию.
- В менеджере есть отдельное действие `Rescan installed platforms`.

## Что умеет менеджер платформ

- просматривать текущий каталог платформ;
- пересканировать установленные платформы;
- добавлять новую платформу вручную;
- менять отображаемое имя;
- менять путь к `1cv8c.exe`;
- удалять запись;
- делать выбранную платформу платформой по умолчанию.

## Как каталог используется в KOT

### Платформа по умолчанию

- `СборкаТекстовСценариев` всегда использует платформу по умолчанию;
- built-in запуск Vanessa использует платформу по умолчанию, если для запуска не выбрана более конкретная платформа;
- `KOT Form Explorer` использует платформу по умолчанию, если для базы не сохранен свой выбор;
- запуск базы из `Infobase Manager` тоже опирается на платформу по умолчанию, если у базы нет сохраненной preferred platform.

### Выбор платформы при запуске

Настройка `kotTestToolkit.platforms.promptForLaunches` управляет тем, спрашивать ли платформу при запуске:

- баз из `Infobase Manager`;
- `KOT Form Explorer`;
- built-in запуска Vanessa.

Если настройка выключена, KOT использует платформу по умолчанию без picker-а.

### Платформа конкретной базы

- Для базы можно отдельно запомнить предпочитаемую платформу в `Infobase Manager`.
- Такой выбор имеет приоритет над общей платформой по умолчанию.
- Это позволяет держать в одном проекте базы на разных версиях платформы.

Подробнее о per-base выборе: [infobase-manager.md](./infobase-manager.md)

## Настройки блока

| Настройка | Назначение |
|---|---|
| `kotTestToolkit.platforms.catalog` | Каталог доступных платформ 1С |
| `kotTestToolkit.platforms.promptForLaunches` | Спрашивать платформу при запуске баз, Form Explorer и built-in Vanessa |

## Что важно учитывать

- KOT ожидает путь именно к `1cv8c.exe`.
- Если файл клиента исчез с диска, запись может остаться в каталоге, но ее нужно будет поправить или удалить вручную.
- Автообнаружение платформ ориентировано на стандартную установку 1С под Windows; нестандартные пути лучше добавлять вручную.
